Core Components of HTML

HTML Tags

Tags are predefined keywords written inside angle brackets < >. Tags tell the browser how to display or structure content on a webpage.

General Syntax:

<tagname> Content </tagname>

Types of Tags:

  • Paired / Container Tags: have both opening and closing tags.
    Example: <p>...</p>
  • Single / Self-Closing Tags: do not have a closing tag.
    Example: <img>, <br>, <hr>
  • Block-Level Tags: start on a new line and take full width.
    Example: <div>, <h1>, <p>
  • Inline Tags: do not start on a new line and take only required space.
    Example: <span>, <a>,
  • Inline-Block Tags: behave like inline, but allow width/height.
    Example: <button>, <input>

HTML Attributes

Attributes provide extra information about an HTML tag. They modify the behavior, appearance, or functionality of an element.

General Syntax:

<tagname attribute="value"> Content </tagname>

Types of Attributes:

  • Global Attributes: usable with any HTML tag.
    Examples: id, class, style, title
  • Element-Specific Attributes: applicable only to certain tags.
    Examples:
    • href → <a>
    • src, alt → <img>
    • type, value → <input>
  • Boolean Attributes: true when present, false when absent.
    Examples: disabled, checked, required
  • Style Attributes: inline CSS for appearance.
    Example: <p style="color:red;">

HTML Elements

Elements are the complete structure formed by an HTML tag, its content, and closing tag.

General Syntax:

<tagname attribute="value"> Content </tagname>

Types of Elements:

  • Block-Level Elements: occupy full width.
    Example: <div>, <section>
  • Inline Elements: occupy only the needed width.
    Example: <span>, <a>
  • Inline-Block Elements: have inline flow but allow dimensions.
    Example: <img>, <button>
  • Empty Elements: no closing tag, no content.
    Example: <br>, <img>
  • Nested Elements: elements inside another element.
    Example: <p>This is <strong>important</strong> text.</p>
